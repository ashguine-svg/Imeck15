1. Introduction

What is Imeck15?

Imeck15 is a tool that recognizes specific images displayed on a PC screen and automatically clicks pre-set locations. It is primarily intended for automating routine, repetitive tasks in PC games and applications.

Key Features

  • Image Recognition: Detects when a registered image (template) appears on the screen.
  • Automatic Clicking: Automatically clicks a specified coordinate or area when an image is detected.
  • Flexible Settings: Allows detailed configuration for each image, such as click interval, recognition accuracy, and click position.
  • Performance Tuning: Equipped with various options to adjust processing speed and PC load according to the operating environment.
  • Folder Function: Manage images in folders, enabling group-based operational control.
  • Drag & Drop: Intuitively rearrange items in the image list or move them into folders.

2. Basic Usage

Here are the four basic steps to get started with Imeck15.

Step 1: Registering Images

Register the images that will serve as the triggers for automatic clicks.

  1. Press the [Capture Image] button on the main screen.
  2. When the "Recognition Area Setting" dialog appears, select [Set Rectangle].
  3. The screen will be covered with a semi-transparent overlay. Drag the mouse to enclose the target you want to click (like a button or icon). Try to make the selection as small as possible while still capturing the unique features.
  4. When the filename input dialog appears, give it a descriptive name and save it.
  5. The image you just registered will be added to the "Registered Images" list on the left.

Step 2: Setting the Recognition Area

Specify the area where Imeck15 will search for images. Limiting this area significantly reduces the load on your PC.

  1. Press the [Set Recognition Area] button at the top of the main screen.
  2. [Set Rectangle]: Select this if you want to search only within a specific region of the entire screen.
  3. [Set Window]: Select this if you want to search only within a specific application window. When you select a window, it automatically calculates the scaling factor based on its standard size, improving image recognition accuracy (details below).

Step 3: Setting the Click Position

Configure where to click when the registered image is found.

  1. Select the desired image from the list on the left.
  2. The image will be displayed in the "Image Preview" tab on the right.
  3. In the "Per-Image Settings" area at the bottom, check the [Point Click] or [Range Click] checkbox.
  4. The mouse cursor will change to a crosshair when over the preview image.
    • Point Click: Click the exact spot you want to target. It will be marked with a red dot.
    • Range Click: Drag the mouse to define a rectangular area. It will be displayed as a blue box. The click will occur somewhere within this area (can also be set to random).

Step 4: Starting and Stopping Monitoring

Once the settings are complete, start the automatic clicking.

  • Start: Press the [Start Monitoring] button in the top-left of the main screen. The status will change to "Monitoring..."
  • Stop: Press the [Stop Monitoring] button.
Shortcuts (Mouse):
  • Right Double-Click: Stop Monitoring
  • Right Triple-Click: Start Monitoring

3. Main Screen Features

Main Control Area

The operation area at the top of the window, which is always visible.

  • [Start Monitoring]/[Stop Monitoring]: Toggles the automatic clicking on and off.
  • [Performance]: Opens the "Performance Monitor," where you can check CPU usage, FPS, etc.
  • [Set Recognition Area]: Sets the area to search for images.
  • [Minimal UI Mode]: Switches to a small window with only essential controls.
  • [Image Folder]: Opens the folder where registered images are stored in Explorer.
  • Status Display: Shows the current state (Idle, Monitoring...).

Registered Images Area (Left)

The area for managing registered images and folders.

  • Image List:
    • Registered images and folders are displayed in a tree format.
    • You can drag & drop items to change their order or move them into folders.
    • You can select multiple items by holding Shift or Ctrl while clicking.
  • [▲ Move Up] / [▼ Move Down]: Moves the selected item up or down in the list.
  • [Add Image]: Adds existing image files (png, jpg, etc.) to the list.
  • [Capture Image]: Captures a new part of the screen to register.
  • [Delete Selected]: Deletes the currently selected items.
  • [Create Folder]: Creates a folder to organize images.
  • [Move In/Out of Folder]: Moves images in and out of folders.
  • Right-click on Folder: Right-clicking a folder opens a menu to set its behavior mode.

Settings & Preview Area (Right)

Access various information and settings by switching tabs.

  • Image Preview: Allows previewing the selected image and setting the click position.
  • Recognition Area: Displays a preview of the currently set recognition area.
  • Log: Displays operation logs.
  • Auto Scale: Configures image scaling settings based on window size.
  • App Settings: Configures the overall behavior of Imeck15.
  • Usage: Displays a detailed guide on per-image settings.
  • Per-Image Settings (Bottom): Configures individual settings for the image selected in the left list.

4. Detailed Functions and Techniques

4.1. Benefits of Setting the Recognition Area

Using [Set Window] to define the recognition area offers two major benefits besides reducing PC load.

1. Automatic Size Adjustment with Window-Based Scaling

Even if the target window's size changes, this feature maintains recognition accuracy by automatically calculating the template image's scale.

How it works:
  • Storing Base Size: The first time you specify a particular window, you are prompted to save its current size as the "base size (1.0x scale)".
  • Automatic Scale Calculation: Subsequently, when you specify the same window, it compares the base size to the current size and automatically calculates the scaling factor (e.g., 1.2x).
  • High-Accuracy Matching: This calculated scale is applied to template matching, ensuring accurate recognition even if the window size has changed.

2. Automatic Window Activation Before Clicking

Just before performing a click, this feature checks if the target window is in the foreground (active) and automatically activates it if it isn't.

How it works:
  • Active State Check: Immediately before clicking, it verifies if the target window is the active foreground window.
  • Forced Activation: If the window is hidden behind another or minimized, it restores the window and brings it to the front before executing the click.
  • Prevents Misfires: This function prevents accidental clicks on other windows, greatly improving the reliability of the automation process.

4.2. Per-Image Setting Items

Configure individual behaviors for the image selected in the list.

  • Recognition Accuracy (Threshold): Sets how similar an image must be to be considered a "match." 0.8 is standard; increasing the value makes the check stricter (1.0 is a perfect match).
  • ROI Enabled (Limit Search Area): When turned on, this speeds up processing by only searching a small, defined area.
    • Fixed: Automatically sets a 200x200 pixel area centered on the click coordinate as the ROI.
    • Variable: Allows you to freely set the search area (ROI) by dragging on the preview screen. Press the [Set ROI Area] button to enable drawing the area on the preview.
  • Backup Click: Functions as a last resort. If no other normal images are found for the specified duration (in seconds), this image will be clicked.
  • Interval (sec): The minimum wait time after a click before the *same image* can be clicked again. This value determines the click priority (a smaller value means higher priority).
  • Debounce (sec): If the same image is targeted for clicking consecutively, this extends the wait time for the second click to "Interval + Debounce Time." This creates an opportunity for other images to be clicked.
  • Click Type: Choose between [Point Click], [Range Click], and [Random in Range].

4.3. How to Control Click Order

If multiple images are found on the screen simultaneously, Imeck15 prioritizes clicking the image with the smallest "Interval" value. You can use this mechanism to control the click priority order.

Example:

  • Set Image A's Interval to 1.5
  • Set Image B's Interval to 2.0

→ Even if A and B are on the screen at the same time, Image A (with the shorter interval) will always be clicked first.

Advanced Technique:
On a screen with multiple buttons that change state (disappear, gray out, etc.) after being clicked, you can automate a sequence of operations by setting the intervals in the order you want them clicked (e.g., 1.5, 2.0, 2.5...).

4.4. Utilizing the Folder Function

Organizing images into folders allows you to control their behavior as a group. Right-click a folder to open its settings window.

  • Normal: Images in the folder are searched as normal.
  • Stop Search (Excluded): All images in the folder are excluded from the search. Useful for temporarily disabling a group of images.
  • Timer Priority: After a specified time (in minutes) passes, the mode switches to searching *only* for images within this folder. Useful for grouping a series of actions that should only run at specific intervals.

Image-Recognition Priority Mode

This is a powerful feature for automating a series of operations that respond to a specific situation (e.g., a pop-up window).

How it works:

  1. Trigger: Priority mode starts if even one image belonging to this folder is found on the screen.
  2. Behavior: While in priority mode, only images within this folder are searched for and clicked.
  3. Deactivation Conditions: The priority mode automatically deactivates and returns to normal monitoring when one of the following conditions is met:
    • All images in the folder have been clicked at least once (Completion).
    • No images from the folder are found for a specified timeout duration (seconds).
Example Usage:
Consider an "Inventory Full" window that appears in a game.
  1. Capture all related images: the "Inventory Full" title, the "OK" button, the "Cancel" button, etc.
  2. Place these images into a new folder named "Handle Inventory Full" and set its mode to "Image-Recognition Priority".
  3. Adjust the intervals of the images within the folder so that the "OK" button is clicked last.
With this setup, the "Handle Inventory Full" folder is normally ignored. But the moment the inventory window appears, Imeck15 detects it, automatically performs the necessary actions (like clicking "OK"), and then returns to normal monitoring after it's done.

4.5. Global Application Settings

Adjust the overall behavior of Imeck15 in the "App Settings" tab.

  • Grayscale Matching: Compares images in black and white. Faster, but cannot distinguish colors.
  • Use DXCam: A faster screen capture method. Effective for capturing game screens, but may not work in all environments.
  • Frame Skip: A higher value lowers the screen check frequency, reducing PC load but also slowing response time.
  • Use OpenCL (GPU Acceleration): Can speed up image processing if a compatible GPU is present.
  • Screen Stability Check: Detects screen rendering or effects and holds clicks until the screen is stable. Adjust the threshold to change the check's sensitivity.
  • Lightweight Mode: Drastically reduces PC load by lowering the screen capture resolution. You can choose from presets (Standard, Performance, Ultra).

5. Performance Optimization

If Imeck15 feels sluggish or causes your PC to lag, review the following settings.

Basic Optimization Steps

Listed in order of effectiveness.

  1. Limit the Recognition Area: This is the most effective method. Use window or rectangle settings to narrow the search area to the bare minimum.
  2. Enable Lightweight Mode: Enable this in the "App Settings" tab. It dramatically improves PC load.
  3. Enable Per-Image ROI: This makes processing very fast as the app only searches in the small area around the click target.
  4. Increase the Frame Skip Value: This is a trade-off with response speed, but it will reduce load.
  5. Use DXCam: If your environment supports it, this provides fast, low-CPU-load capturing.

About Eco Mode

The [Eco Mode] in the "App Settings" tab is a feature to further reduce PC load.

How it works:
  • Activation: Automatically activates when no recognizable image is found for 5 seconds after the last click.
  • Behavior: When activated, the screen check frequency drops to about once per second, significantly reducing CPU usage.
  • Deactivation: If an image is recognized and a click is performed during Eco Mode, it automatically deactivates and returns to the normal monitoring frequency.
This feature is particularly useful when automating tasks with long wait times. It is enabled by default, but you can turn it off if you always require the fastest possible response.

6. Various Windows

Performance Monitor

Opened via the [Performance] button on the main screen. It allows you to check CPU/memory usage, FPS (frame checks per second), total clicks, and other info in real-time. You can also start/stop monitoring or set the recognition area using the buttons at the top. Shortcuts are available: Right double-click to stop monitoring, right triple-click to start monitoring.

Minimal UI Mode

Switched via the [Minimal UI Mode] button. This hides the main window and displays a small window with essential controls like start/stop monitoring.

7. Frequently Asked Questions (FAQ)

Q: Images are not being found correctly.

A: Please check the following points:

  • Is the "Recognition Area" set correctly?
  • Is the "Recognition Accuracy" in "Per-Image Settings" too high? Try lowering it slightly.
  • Has the window size of your game or application changed? Review the settings in the "Auto Scale" tab or try re-capturing the image.

Q: The app is heavy and my PC is lagging.

A: Please refer to section "5. Performance Optimization" above and review your settings.

Q: It clicks in the middle of a screen transition.

A: Check if "Screen Stability Check" is enabled in the "App Settings" tab. If it is, try lowering the "Threshold" value slightly to make the check stricter.
