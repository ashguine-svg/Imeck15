Руководство по настройке кликов для каждого изображения в Imeck15

Это руководство объясняет, как настроить действия клика, когда найдено зарегистрированное изображение. Оно в первую очередь предназначено для автоматизации операций в оконных приложениях.

1. Как зарегистрировать изображения для клика

Сначала зарегистрируйте изображение, которое будет служить целью клика. Использование функции «Захватить изображение» является основным методом.
Совет: Рекомендуется захватывать небольшие части, такие как кнопки или значки, в качестве цели.

  • Причина 1: Более быстрая обработка
    Поиск небольшого изображения менее требователен к ПК, чем поиск по всему экрану.
  • Причина 2: Точный клик
    Даже если на экране существует несколько одинаковых кнопок, захват только определенной части позволяет точно кликнуть по предполагаемой цели.

2. Как контролировать порядок кликов (Настройка интервала)

«Интервал» — это минимальное время ожидания (в секундах) после клика по изображению, прежде чем по этому же изображению можно будет кликнуть снова.

[Важно] Механизм приоритета клика
Если во время мониторинга на экране обнаружено несколько кликабельных изображений, Imeck15 отдает приоритет клику по тому, у которого самая короткая настройка «Интервала», и не кликает по другим изображениям. Используйте этот механизм для управления последовательностью кликов.

3. Техника клика по нескольким местам на одном экране

Применяя механизм «Настройки интервала», вы можете заставить его последовательно кликать по нескольким местам на одном экране.
Предварительное условие: Изображение или текст в месте клика должны измениться (исчезнуть, стать серым и т. д.) после клика.

Пример шагов настройки:

  1. Зарегистрируйте части A, B и C, по которым вы хотите кликнуть на экране, как отдельные изображения.
  2. Установите более короткое время интервала для изображений, по которым вы хотите кликнуть раньше. (например, A: 1.5 сек, B: 2.0 сек, C: 2.5 сек)
  3. Начните мониторинг. Сначала будет клик по части A с самым коротким интервалом.
  4. После клика, если часть A исчезнет с экрана, часть B станет следующей целью клика в последующем цикле мониторинга.
  5. Аналогично, часть C будет кликнута последней.

4. Как настроить точность и диапазон распознавания

  • Порог распознавания:
    Эта настройка определяет, насколько схожими должны быть изображения, чтобы считаться «совпадением». Значения около 0.8 обычно достаточно, но установка на 0.9 или выше может быть эффективна для строгого различения незначительных отличий, таких как различия в тексте.
  • Диапазон поиска (настройка ROI):
    Включение «ROI Включен» ограничивает цель поиска областью 200x200 пикселей, центрированной вокруг координаты клика. Это значительно ускоряет обработку и сильно снижает нагрузку на ПК. (Режим переменного ROI позволяет свободно устанавливать диапазон).
  • Спецификация диапазона ROI:
    Вместо диапазона 200x200, центрированного по положению клика, эта функция позволяет вам указать определенную область для фокусировки и назначить отдельную позицию клика.

5. Настройка «Устранение дребезга (Debounce)» для особых ситуаций

«Устранение дребезга» используется для сложных операций, таких как «кликнуть по изображению A с коротким интервалом, затем кликнуть по другому изображению B, и после короткой паузы снова кликнуть по изображению A, чтобы перейти к следующему экрану».

Механизм: Установка времени устранения дребезга продлевает время ожидания для второго клика до «Интервал + Время устранения дребезга», когда то же изображение является целью последовательно. Это создает возможность для клика по другим изображениям в первую очередь.
