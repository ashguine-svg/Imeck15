Imeck15 이미지별 클릭 설정 가이드

이 가이드는 등록된 이미지가 발견되었을 때 클릭 동작을 구성하는 방법을 설명합니다. 주로 창 모드 애플리케이션의 작업을 자동화하기 위해 만들어졌습니다.

1. 클릭할 이미지를 등록하는 방법

먼저, 클릭 대상이 될 이미지를 등록합니다. "이미지 캡처" 기능을 사용하는 것이 기본 방법입니다.
팁: 버튼이나 아이콘과 같은 작은 부분을 대상으로 캡처하는 것이 좋습니다.

  • 이유 1: 더 빠른 처리
    작은 이미지를 검색하는 것이 전체 화면을 검색하는 것보다 PC에 부하가 적습니다.
  • 이유 2: 정확한 클릭
    화면에 동일한 버튼이 여러 개 존재하더라도, 특정 부분만 캡처하면 의도한 대상을 정확하게 클릭할 수 있습니다.

2. 클릭 순서를 제어하는 방법 (간격 설정)

"간격(Interval)"은 이미지를 클릭한 후 동일한 이미지를 다시 클릭할 수 있을 때까지의 최소 대기 시간(초)입니다.

[중요] 클릭 우선순위 메커니즘
모니터링 중에 화면에서 여러 개의 클릭 가능한 이미지가 발견되면, Imeck15는 가장 짧은 "간격" 설정을 가진 이미지 클릭을 우선시하고 다른 이미지는 클릭하지 않습니다. 이 메커니즘을 사용하여 클릭 순서를 제어하십시오.

3. 한 화면에서 여러 위치를 클릭하는 기술

"간격 설정" 메커니즘을 적용하면 한 화면에서 여러 위치를 순차적으로 클릭하도록 만들 수 있습니다.
전제 조건: 클릭된 위치의 이미지나 텍스트가 클릭된 후 변경되어야 합니다(사라지거나, 회색으로 변하거나 등).

예시 설정 단계:

  1. 화면에서 클릭하려는 부분 A, B, C를 별도의 이미지로 등록합니다.
  2. 더 일찍 클릭하려는 이미지에 대해 더 짧은 간격 시간을 설정합니다. (예: A: 1.5초, B: 2.0초, C: 2.5초)
  3. 모니터링을 시작합니다. 먼저, 가장 짧은 간격을 가진 부분 A가 클릭됩니다.
  4. 클릭 후, 부분 A가 화면에서 사라지면, 부분 B가 다음 모니터링 주기에서 다음 클릭 대상이 됩니다.
  5. 마찬가지로, 부분 C가 가장 마지막에 클릭됩니다.

4. 인식 정확도 및 범위를 조정하는 방법

  • 인식 임계값:
    이 설정은 이미지가 "일치"로 간주되기 위해 얼마나 유사해야 하는지를 결정합니다. 일반적으로 0.8 정도의 값이 충분하지만, 텍스트 변화와 같은 미세한 차이를 엄격하게 구별하려면 0.9 이상으로 설정하는 것이 효과적일 수 있습니다.
  • 검색 범위 (ROI 설정):
    "ROI 활성화"를 활성화하면 검색 대상을 클릭 좌표를 중심으로 한 200x200 픽셀 영역으로 제한합니다. 이는 처리 속도를 크게 높이고 PC의 부하를 크게 줄입니다. (가변 ROI 모드는 범위를 자유롭게 설정할 수 있도록 합니다).
  • ROI 범위 지정:
    클릭 위치를 중심으로 한 200x200 범위 대신, 이 기능은 집중할 특정 영역을 지정하고 별도의 클릭 위치를 지정할 수 있도록 합니다.

5. 특별한 상황을 위한 "디바운스(Debounce)" 설정

"디바운스"는 "이미지 A를 짧은 간격으로 클릭한 다음, 다른 이미지 B를 클릭하고, 짧은 멈춤 후, 다음 화면으로 진행하기 위해 이미지 A를 다시 클릭하는" 것과 같은 복잡한 작업에 사용됩니다.

메커니즘: 디바운스 시간을 설정하면 동일한 이미지가 연속적으로 대상이 될 때 두 번째 클릭에 대한 대기 시간을 "간격 + 디바운스 시간"으로 연장합니다. 이는 다른 이미지가 먼저 클릭될 기회를 만듭니다.
