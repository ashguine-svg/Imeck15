Imeck15 개별 이미지 클릭 설정 가이드

이 가이드에서는 등록된 이미지를 찾았을 때 클릭 동작을 구성하는 방법을 설명합니다. 주로 창 응용 프로그램의 작업을 자동화하기 위한 것입니다.

1. 클릭할 이미지 등록 방법

먼저 클릭 대상이 될 이미지를 등록합니다. "이미지 캡처" 기능을 사용하는 것이 기본 방법입니다.
팁: 버튼이나 아이콘과 같은 작은 부분을 대상으로 캡처하는 것이 좋습니다.

  • 이유 1: 처리 속도 향상
    전체 화면을 검색하는 것보다 작은 이미지를 검색하는 것이 PC에 부담이 적습니다.
  • 이유 2: 정확한 클릭
    화면에 동일한 버튼이 여러 개 있더라도 특정 부분만 캡처하면 의도한 대상을 정확하게 클릭할 수 있습니다.

2. 클릭 순서 제어 방법 (간격 설정)

"간격"은 이미지를 클릭한 후 동일한 이미지를 다시 클릭할 수 있을 때까지의 최소 대기 시간(초)입니다.

[중요] 클릭 우선 순위 메커니즘
모니터링 중 화면에서 여러 개의 클릭 가능한 이미지가 발견되면 Imeck15는 "간격" 설정이 가장 짧은 이미지를 우선적으로 클릭하고 다른 이미지는 클릭하지 않습니다. 이 메커니즘을 사용하여 클릭 순서를 제어하십시오.

3. 한 화면에서 여러 위치 클릭 기법

"간격 설정" 메커니즘을 적용하여 한 화면에서 여러 위치를 순차적으로 클릭하도록 할 수 있습니다.
전제 조건: 클릭 후 클릭된 위치의 이미지나 텍스트가 변경되어야 합니다 (사라지거나 회색으로 변하는 등).

예제 설정 단계:

  1. 화면에서 클릭하려는 부분 A, B, C를 별도의 이미지로 등록합니다.
  2. 먼저 클릭하려는 이미지에 더 짧은 간격 시간을 설정합니다. (예: A: 1.5초, B: 2.0초, C: 2.5초)
  3. 모니터링을 시작합니다. 먼저 간격이 가장 짧은 부분 A가 클릭됩니다.
  4. 클릭 후, 부분 A가 화면에서 사라지면 후속 모니터링 주기에서 부분 B가 다음 클릭 대상이 됩니다.
  5. 마찬가지로, 마지막으로 부분 C가 클릭됩니다.

4. 인식 정확도 및 범위 조정 방법

  • 인식 임계값:
    이 설정은 이미지가 "일치"로 간주되기 위해 얼마나 유사해야 하는지를 결정합니다. 일반적으로 0.8 정도의 값이면 충분하지만, 텍스트 변형과 같은 미세한 차이를 엄격하게 구별하려면 0.9 이상으로 설정하는 것이 효과적일 수 있습니다.
  • 검색 범위 (ROI 설정):
    "ROI 활성화"를 사용하면 검색 대상을 클릭 좌표를 중심으로 한 200x200 픽셀 영역으로 제한합니다. 이는 처리 속도를 크게 향상시키고 PC 부하를 크게 줄입니다. (가변 ROI 모드를 사용하면 범위를 자유롭게 설정할 수 있습니다).
  • ROI 범위 지정:
    클릭 위치를 중심으로 한 200x200 범위 대신, 이 기능을 사용하면 집중할 특정 영역을 지정하고 별도의 클릭 위치를 지정할 수 있습니다.

5. 특수 상황을 위한 "디바운스" 설정

"디바운스"는 "짧은 간격으로 이미지 A를 클릭한 다음 다른 이미지 B를 클릭하고, 잠시 멈춘 후 다시 이미지 A를 클릭하여 다음 화면으로 진행"과 같은 복잡한 작업에 사용됩니다.

메커니즘: 디바운스 시간을 설정하면 동일한 이미지가 연속적으로 타겟팅될 때 두 번째 클릭 대기 시간이 "간격 + 디바운스 시간"으로 연장됩니다. 이는 다른 이미지가 먼저 클릭될 기회를 만듭니다.
