1. 소개

Imeck15란?

Imeck15는 PC 화면에 표시되는 특정 이미지를 인식하여 미리 설정된 위치를 자동으로 클릭하는 도구입니다. 주로 PC 게임 및 응용 프로그램에서 일상적이고 반복적인 작업을 자동화하기 위한 것입니다.

주요 기능

  • 이미지 인식: 등록된 이미지(템플릿)가 화면에 나타나는 시점을 감지합니다.
  • 자동 클릭: 이미지가 감지되면 지정된 좌표 또는 영역을 자동으로 클릭합니다.
  • 유연한 설정: 클릭 간격, 인식 정확도, 클릭 위치 등 각 이미지에 대한 상세한 구성이 가능합니다.
  • 성능 튜닝: 운영 환경에 따라 처리 속도와 PC 부하를 조정할 수 있는 다양한 옵션을 갖추고 있습니다.
  • 폴더 기능: 이미지를 폴더로 관리하여 그룹 기반의 운영 제어를 가능하게 합니다.
  • 드래그 앤 드롭: 이미지 목록의 항목을 직관적으로 재정렬하거나 폴더로 이동할 수 있습니다.

2. 기본 사용법

Imeck15를 시작하기 위한 네 가지 기본 단계입니다.

1단계: 이미지 등록

자동 클릭의 트리거 역할을 할 이미지를 등록합니다.

  1. 메인 화면에서 [이미지 캡처] 버튼을 누릅니다.
  2. "인식 영역 설정" 대화 상자가 나타나면 [사각형 설정]을 선택합니다.
  3. 화면이 반투명 오버레이로 덮입니다. 클릭하려는 대상(버튼 또는 아이콘 등) 주위로 마우스를 드래그하여 선택합니다. 고유한 특징을 캡처하면서 선택 영역을 가능한 한 작게 만드십시오.
  4. 파일명 입력 대화 상자가 나타나면 설명적인 이름을 지정하고 저장합니다.
  5. 방금 등록한 이미지가 왼쪽의 "등록된 이미지" 목록에 추가됩니다.

2단계: 인식 영역 설정

Imeck15가 이미지를 검색할 영역을 지정합니다. 이 영역을 제한하면 PC 부하를 크게 줄일 수 있습니다.

  1. 메인 화면 상단의 [인식 영역 설정] 버튼을 누릅니다.
  2. [사각형 설정]: 전체 화면의 특정 영역 내에서만 검색하려는 경우 선택합니다.
  3. [창 설정]: 특정 응용 프로그램 창 내에서만 검색하려는 경우 선택합니다. 창을 선택하면 표준 크기를 기준으로 배율을 자동으로 계산하여 이미지 인식 정확도를 향상시킵니다 (자세한 내용은 아래 참조).

3단계: 클릭 위치 설정

등록된 이미지를 찾았을 때 클릭할 위치를 구성합니다.

  1. 왼쪽 목록에서 원하는 이미지를 선택합니다.
  2. 이미지가 오른쪽의 "이미지 미리보기" 탭에 표시됩니다.
  3. 하단의 "개별 이미지 설정" 영역에서 [점 클릭] 또는 [범위 클릭] 체크박스를 선택합니다.
  4. 미리보기 이미지 위에서 마우스 커서가 십자 모양으로 변경됩니다.
    • 점 클릭: 타겟팅하려는 정확한 지점을 클릭합니다. 빨간 점으로 표시됩니다.
    • 범위 클릭: 마우스를 드래그하여 사각형 영역을 정의합니다. 파란색 상자로 표시됩니다. 클릭은 이 영역 내 어딘가에서 발생합니다 (무작위로 설정할 수도 있음).

4단계: 모니터링 시작 및 중지

설정이 완료되면 자동 클릭을 시작합니다.

  • 시작: 메인 화면 왼쪽 상단의 [모니터링 시작] 버튼을 누릅니다. 상태가 "모니터링 중..."으로 변경됩니다.
  • 중지: [모니터링 중지] 버튼을 누릅니다.
단축키 (마우스):
  • 오른쪽 더블클릭: 모니터링 중지
  • 오른쪽 트리플클릭: 모니터링 시작

3. 메인 화면 기능

메인 컨트롤 영역

창 상단에 항상 표시되는 작업 영역입니다.

  • [모니터링 시작]/[모니터링 중지]: 자동 클릭을 켜고 끕니다.
  • [성능]: CPU 사용량, FPS 등을 확인할 수 있는 "성능 모니터"를 엽니다.
  • [인식 영역 설정]: 이미지를 검색할 영역을 설정합니다.
  • [최소 UI 모드]: 필수 컨트롤만 있는 작은 창으로 전환합니다.
  • [이미지 폴더]: 등록된 이미지가 저장된 폴더를 탐색기에서 엽니다.
  • 상태 표시: 현재 상태(대기 중, 모니터링 중...)를 표시합니다.

등록된 이미지 영역 (왼쪽)

등록된 이미지 및 폴더를 관리하는 영역입니다.

  • 이미지 목록:
    • 등록된 이미지와 폴더가 트리 형식으로 표시됩니다.
    • 항목을 드래그 앤 드롭하여 순서를 변경하거나 폴더로 이동할 수 있습니다.
    • Shift 또는 Ctrl 키를 누른 상태에서 클릭하여 여러 항목을 선택할 수 있습니다.
  • [▲ 위로] / [▼ 아래로]: 선택한 항목을 목록에서 위 또는 아래로 이동합니다.
  • [이미지 추가]: 기존 이미지 파일(png, jpg 등)을 목록에 추가합니다.
  • [이미지 캡처]: 화면의 새 부분을 캡처하여 등록합니다.
  • [선택 항목 삭제]: 현재 선택된 항목을 삭제합니다.
  • [폴더 생성]: 이미지를 정리할 폴더를 만듭니다.
  • [폴더로 이동/폴더 밖으로 이동]: 이미지를 폴더 안팎으로 이동합니다.
  • 폴더 오른쪽 클릭: 폴더를 오른쪽 클릭하면 해당 폴더의 동작 모드를 설정하는 메뉴가 열립니다.

설정 및 미리보기 영역 (오른쪽)

탭을 전환하여 다양한 정보와 설정에 액세스합니다.

  • 이미지 미리보기: 선택한 이미지를 미리 보고 클릭 위치를 설정할 수 있습니다.
  • 인식 영역: 현재 설정된 인식 영역의 미리보기를 표시합니다.
  • 로그: 작업 로그를 표시합니다.
  • 자동 스케일: 창 크기에 따른 이미지 스케일링 설정을 구성합니다.
  • 앱 설정: Imeck15의 전체적인 동작을 구성합니다.
  • 사용법: 개별 이미지 설정에 대한 상세 가이드를 표시합니다.
  • 개별 이미지 설정 (하단): 왼쪽 목록에서 선택한 이미지의 개별 설정을 구성합니다.

4. 상세 기능 및 기법

4.1. 인식 영역 설정의 이점

[창 설정]을 사용하여 인식 영역을 정의하면 PC 부하 감소 외에 두 가지 주요 이점이 있습니다.

1. 창 기반 스케일링을 통한 자동 크기 조정

대상 창의 크기가 변경되더라도 이 기능은 템플릿 이미지의 스케일을 자동으로 계산하여 인식 정확도를 유지합니다.

작동 방식:
  • 기본 크기 저장: 특정 창을 처음 지정할 때 현재 크기를 "기본 크기(1.0배 스케일)"로 저장하라는 메시지가 표시됩니다.
  • 자동 스케일 계산: 이후 동일한 창을 지정하면 기본 크기와 현재 크기를 비교하여 스케일링 팩터(예: 1.2배)를 자동으로 계산합니다.
  • 고정밀 매칭: 이 계산된 스케일이 템플릿 매칭에 적용되어 창 크기가 변경되더라도 정확한 인식을 보장합니다.

2. 클릭 전 자동 창 활성화

클릭을 수행하기 직전에 이 기능은 대상 창이 전경(활성)에 있는지 확인하고 그렇지 않은 경우 자동으로 활성화합니다.

작동 방식:
  • 활성 상태 확인: 클릭 직전에 대상 창이 활성 전경 창인지 확인합니다.
  • 강제 활성화: 창이 다른 창 뒤에 숨겨져 있거나 최소화된 경우 클릭을 실행하기 전에 창을 복원하고 전면으로 가져옵니다.
  • 오작동 방지: 이 기능은 다른 창에 대한 우발적인 클릭을 방지하여 자동화 프로세스의 신뢰성을 크게 향상시킵니다.

4.2. 개별 이미지 설정 항목

목록에서 선택한 이미지에 대한 개별 동작을 구성합니다.

  • 인식 정확도 (임계값): 이미지가 "일치"로 간주되기 위해 얼마나 유사해야 하는지 설정합니다. 0.8이 표준이며, 값을 높이면 검사가 더 엄격해집니다(1.0은 완벽 일치).
  • ROI 활성화 (검색 영역 제한): 켜면 작고 정의된 영역만 검색하여 처리를 가속화합니다.
    • 고정: 클릭 좌표를 중심으로 한 200x200 픽셀 영역을 ROI로 자동 설정합니다.
    • 가변: 미리보기 화면에서 드래그하여 검색 영역(ROI)을 자유롭게 설정할 수 있습니다. [ROI 영역 설정] 버튼을 눌러 미리보기에서 영역 그리기를 활성화합니다.
  • 백업 클릭: 최후의 수단으로 작동합니다. 지정된 기간(초) 동안 다른 일반 이미지를 찾지 못하면 이 이미지가 클릭됩니다.
  • 간격 (초): 클릭 후 *동일한 이미지*를 다시 클릭할 수 있을 때까지의 최소 대기 시간입니다. 이 값은 클릭 우선 순위를 결정합니다(값이 작을수록 우선 순위가 높음).
  • 디바운스 (초): 동일한 이미지가 연속적으로 클릭 대상으로 지정될 경우, 두 번째 클릭 대기 시간을 "간격 + 디바운스 시간"으로 연장합니다. 이는 다른 이미지가 클릭될 기회를 만듭니다.
  • 클릭 유형: [점 클릭], [범위 클릭], [범위 내 무작위] 중에서 선택합니다.

4.3. 클릭 순서 제어 방법

화면에 여러 이미지가 동시에 발견되면 Imeck15는 "간격" 값이 가장 작은 이미지를 우선적으로 클릭합니다. 이 메커니즘을 사용하여 클릭 우선 순위 순서를 제어할 수 있습니다.

예:

  • 이미지 A의 간격을 1.5로 설정
  • 이미지 B의 간격을 2.0으로 설정

→ 화면에 A와 B가 동시에 있더라도 이미지 A(간격이 짧음)가 항상 먼저 클릭됩니다.

고급 기법:
클릭 후 상태가 변경되는(사라지거나 회색으로 변하는 등) 여러 버튼이 있는 화면에서는 클릭하려는 순서대로 간격을 설정하여(예: 1.5, 2.0, 2.5...) 일련의 작업을 자동화할 수 있습니다.

4.4. 폴더 기능 활용

이미지를 폴더로 정리하면 그룹으로 동작을 제어할 수 있습니다. 폴더를 오른쪽 클릭하여 설정 창을 엽니다.

  • 정상: 폴더 내 이미지는 정상적으로 검색됩니다.
  • 검색 중지 (제외됨): 폴더 내 모든 이미지는 검색에서 제외됩니다. 일시적으로 이미지 그룹을 비활성화하는 데 유용합니다.
  • 타이머 우선: 지정된 시간(분)이 지나면 이 폴더 내의 이미지*만* 검색하는 모드로 전환됩니다. 특정 간격으로만 실행해야 하는 일련의 작업을 그룹화하는 데 유용합니다.

이미지 인식 우선 모드

특정 상황(예: 팝업 창)에 대응하는 일련의 작업을 자동화하는 데 매우 강력한 기능입니다.

작동 방식:

  1. 트리거: 화면에서 이 폴더에 속한 이미지가 하나라도 발견되면 우선 모드가 시작됩니다.
  2. 동작: 우선 모드 중에는 이 폴더 내의 이미지만 검색되고 클릭됩니다.
  3. 비활성화 조건: 다음 조건 중 하나가 충족되면 우선 모드가 자동으로 비활성화되고 일반 모니터링으로 돌아갑니다:
    • 폴더 내 모든 이미지가 최소 한 번 클릭되었습니다 (완료).
    • 지정된 시간 초과 기간(초) 동안 해당 폴더의 이미지가 발견되지 않았습니다.
사용 예:
게임에서 나타나는 "인벤토리 가득 참" 창을 고려해 봅시다.
  1. 모든 관련 이미지 캡처: "인벤토리 가득 참" 제목, "확인" 버튼, "취소" 버튼 등.
  2. 이 이미지들을 "인벤토리 처리"라는 새 폴더에 넣고 모드를 "이미지 인식 우선"으로 설정합니다.
  3. 폴더 내 이미지의 간격을 조정하여 "확인" 버튼이 마지막에 클릭되도록 합니다.
이 설정을 사용하면 "인벤토리 처리" 폴더는 일반적으로 무시됩니다. 하지만 인벤토리 창이 나타나는 순간 Imeck15가 이를 감지하여 필요한 작업(예: "확인" 클릭)을 자동으로 수행한 다음 완료 후 일반 모니터링으로 돌아갑니다.

4.5. 전역 애플리케이션 설정

"앱 설정" 탭에서 Imeck15의 전체적인 동작을 조정합니다.

  • 그레이스케일 매칭: 이미지를 흑백으로 비교합니다. 더 빠르지만 색상을 구별할 수 없습니다.
  • DXCam 사용: 더 빠른 화면 캡처 방법입니다. 게임 화면 캡처에 효과적이지만 모든 환경에서 작동하지 않을 수 있습니다.
  • 프레임 건너뛰기: 값이 높을수록 화면 확인 빈도가 낮아져 PC 부하가 줄어들지만 응답 시간도 느려집니다.
  • OpenCL 사용 (GPU 가속): 호환되는 GPU가 있는 경우 이미지 처리 속도를 높일 수 있습니다.
  • 화면 안정성 검사: 화면 렌더링 또는 효과를 감지하고 화면이 안정될 때까지 클릭을 보류합니다. 임계값을 조정하여 검사 감도를 변경합니다.
  • 경량 모드: 화면 캡처 해상도를 낮춰 PC 부하를 대폭 줄입니다. 프리셋(표준, 성능, 울트라) 중에서 선택할 수 있습니다.

5. 성능 최적화

Imeck15가 느리게 느껴지거나 PC가 지연되는 경우 다음 설정을 검토하십시오.

기본 최적화 단계

효과 순서대로 나열되어 있습니다.

  1. 인식 영역 제한: 가장 효과적인 방법입니다. 창 또는 사각형 설정을 사용하여 검색 영역을 최소한으로 좁히십시오.
  2. 경량 모드 활성화: "앱 설정" 탭에서 이 기능을 활성화합니다. PC 부하를 극적으로 개선합니다.
  3. 개별 이미지 ROI 활성화: 앱이 클릭 대상 주변의 작은 영역만 검색하므로 처리가 매우 빨라집니다.
  4. 프레임 건너뛰기 값 증가: 응답 속도와의 절충안이지만 부하를 줄일 수 있습니다.
  5. DXCam 사용: 환경이 지원하는 경우 빠르고 CPU 부하가 낮은 캡처를 제공합니다.

절전 모드 정보

"앱 설정" 탭의 [절전 모드]는 PC 부하를 더욱 줄이기 위한 기능입니다.

작동 방식:
  • 활성화: 마지막 클릭 후 5초 동안 인식 가능한 이미지가 발견되지 않으면 자동으로 활성화됩니다.
  • 동작: 활성화되면 화면 확인 빈도가 초당 약 1회로 떨어져 CPU 사용량을 크게 줄입니다.
  • 비활성화: 절전 모드 중 이미지가 인식되고 클릭이 수행되면 자동으로 비활성화되고 일반 모니터링 빈도로 돌아갑니다.
이 기능은 대기 시간이 긴 작업을 자동화할 때 특히 유용합니다. 기본적으로 활성화되어 있지만 항상 가장 빠른 응답이 필요한 경우 끌 수 있습니다.
