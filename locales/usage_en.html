Imeck15 Per-Image Click Settings Guide

This guide explains how to configure click actions when a registered image is found. It is primarily intended for automating operations in windowed applications.

1. How to Register Images for Clicking

First, register the image that will serve as the click target. Using the "Capture Image" function is the basic method.
Tip: It is recommended to capture small parts like buttons or icons as the target.

  • Reason 1: Faster Processing
    Searching for a small image is less demanding on the PC than searching the entire screen.
  • Reason 2: Accurate Clicking
    Even if multiple identical buttons exist on the screen, capturing only a specific part allows for precise clicking on the intended target.

2. How to Control Click Order (Interval Setting)

The "Interval" is the minimum waiting time (in seconds) after clicking an image before the same image can be clicked again.

[Important] Click Priority Mechanism
If multiple clickable images are found on the screen during monitoring, Imeck15 prioritizes clicking the one with the shortest "Interval" setting and does not click other images. Use this mechanism to control the click sequence.

3. Technique for Clicking Multiple Locations on One Screen

By applying the "Interval Setting" mechanism, you can make it click multiple locations sequentially on a single screen.
Prerequisite: The image or text at the clicked location must change (disappear, gray out, etc.) after being clicked.

Example Setup Steps:

  1. Register parts A, B, and C that you want to click on the screen as separate images.
  2. Set shorter interval times for images you want to click earlier. (e.g., A: 1.5 sec, B: 2.0 sec, C: 2.5 sec)
  3. Start monitoring. First, part A with the shortest interval will be clicked.
  4. After clicking, if part A disappears from the screen, part B will become the next click target in the subsequent monitoring cycle.
  5. Similarly, part C will be clicked last.

4. How to Adjust Recognition Accuracy and Range

  • Recognition Threshold:
    This setting determines how similar images need to be to be considered a "match". A value around 0.8 is usually sufficient, but setting it to 0.9 or higher can be effective for strictly distinguishing minor differences like text variations.
  • Search Range (ROI Setting):
    Enabling "ROI Enabled" limits the search target to a 200x200 pixel area centered around the click coordinate. This significantly speeds up processing and greatly reduces the load on the PC. (Variable ROI mode allows setting the range freely).
  • ROI Range Specification:
    Instead of a 200x200 range centered on the click position, this feature allows you to specify a specific area to focus on and designate a separate click position.

5. "Debounce" Setting for Special Situations

"Debounce" is used for complex operations like "click image A with a short interval, then click another image B, and after a short pause, click image A again to proceed to the next screen."

Mechanism: Setting a debounce time extends the waiting period for the second click to "Interval + Debounce Time" when the same image is targeted consecutively. This creates an opportunity for other images to be clicked first.
